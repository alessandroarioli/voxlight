English

## Description

Voxlight turns any web page into an audiobook — with a bouncing-ball highlight that follows the words as they're spoken.

HOW IT WORKS
1. Select any text on a page
2. Right-click → "🔊 Read aloud with Voxlight" (or press Alt+Shift+S)
3. Listen — the current word lights up and already-read text stays tinted, so your eyes never lose the place

FEATURES
• Follow-along highlight — word-by-word, like karaoke for reading
• Auto-scroll keeps the spoken text in view
• Floating pause / resume / stop control (Esc also stops)
• Choose any system voice, adjust speed and pitch
• Works on articles, docs, emails — anywhere you can select text

PRIVATE BY DESIGN
• 100% on-device: speech is generated by your browser's built-in engine
• Selected text NEVER leaves your computer
• No accounts, no analytics, no tracking, no ads
• Works offline with local voices

WHY VOXLIGHT
Perfect for proofreading your own writing, resting your eyes on long reads, language learning (see + hear simultaneously), and accessibility.

TIP: On macOS, download free "Premium" voices in System Settings → Accessibility → Spoken Content for dramatically better speech quality. Voices marked "· local" in Voxlight's settings give the most precise highlight tracking.

## Store form answers

### Single purpose description
Voxlight reads aloud the text the user selects on a web page, highlighting each word as it is spoken.

### Permission justifications

**activeTab** — Grants access to the current tab only when the user explicitly invokes Voxlight (right-click menu or keyboard shortcut), so the selected text can be read aloud on that page.

**scripting** — Injects the reading/highlighting script into the current tab on demand, only after the user invokes Voxlight. No scripts run in the background or on pages where the user hasn't asked for reading.

**contextMenus** — Adds the "Read aloud with Voxlight" item to the right-click menu on text selections. This is the primary way users start the feature.

**storage** — Saves the user's voice, speed, and pitch preferences so they persist between sessions.

### Remote code
No, this extension does not use remote code. All code is packaged in the extension.

### Data usage
- Does NOT collect any user data (check "no data collected" for every category).
- Text-to-speech runs locally via the browser's Web Speech API; selected text is never transmitted.
